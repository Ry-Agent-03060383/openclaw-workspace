## 1. 项目概述

### 1.1 项目背景

随着金融科技的快速发展，传统金融服务模式面临数字化转型压力。中小企业融资难、风控效率低、数据孤岛等问题日益突出。本项目旨在构建一站式智慧金融服务平台，通过整合企业信用数据、自动化贷款审批流程、智能风险评估和可视化数据分析，提升金融服务效率和用户体验。

### 1.2 项目目标

- **业务目标**: 实现企业信用查询、贷款申请、风险评估全流程线上化
- **技术目标**: 构建高可用、可扩展的微服务架构
- **用户目标**: 为企业客户提供便捷的融资服务，为金融机构提供高效的 风控工具

### 1.3 目标用户

| 用户类型 | 描述 | 主要需求 |
|---------|------|---------|
| 中小企业 | 有融资需求的企业 | 快速获取信用报告、申请贷款 |
| 金融机构 | 银行、贷款公司 | 高效审批、风险管控 |
| 风控人员 | 内部风控团队 | 实时监控、自动预警 |
| 管理人员 | 平台运营方 | 数据分析、决策支持 |

## 3. MoSCoW 优先级定义

### 3.1 优先级分级标准

- **Must (必须完成)**: 核心业务功能，上线必备，阻塞性
- **Should (应该完成)**: 重要功能，提升体验，上线前完成
- **Could (可以完成)**: 增强功能，有时间再实现
- **Won't (不会完成)**: 本期不包含，待未来规划

### 3.2 功能优先级矩阵

#### MUST (必须完成) - 核心基础

| 功能 | 描述 | 验收标准 |
|-----|------|---------|
| M1.1 企业信息查询 | 按统一社会信用代码/企业名称查询 | 支持模糊搜索，返回基本信息 |
| M1.2 工商数据接入 | 对接第三方工商数据源 | 数据准确率>95% |
| M4.1 贷款申请入口 | 用户注册登录后创建贷款申请 | 支持PC端操作 |
| M4.2 申请材料上传 | 支持营业执照、财务报表等上传 | 支持PDF/图片格式 |
| M5.1 基础审批规则 | 配置化规则引擎 | 支持10+审批规则配置 |
| M6.1 信用评分模型 | 基于多维度计算信用分 | 输出0-100信用分 |
| M6.2 风险等级划分 | 自动划分风险等级(AAA/AA/A/BBB/BB/B/C) | 与评分模型联动 |
| M8.1 站内通知 | 申请状态变更通知 | 实时推送 |

#### SHOULD (应该完成) - 重要功能

| 功能 | 描述 | 验收标准 |
|-----|------|---------|
| M2.1 标准信用报告 | 生成PDF信用报告 | 包含企业概况、信用评分、风险提示 |
| M3.1 产品列表展示 | 展示可申请贷款产品 | 支持筛选、排序 |
| M5.2 人工审批入口 | 人工复审/终审界面 | 支持通过/拒绝/退回 |
| M7.1 管理仪表盘 | 核心业务数据展示 | 包含申请量、审批通过率、放款额 |
| M6.3 风险预警 | 异常行为自动预警 | 支持邮件/短信通知风控人员 |

#### COULD (可以完成) - 增强功能

| 功能 | 描述 | 验收标准 |
|-----|------|---------|
| M1.3 历史信用记录 | 查看企业历史信用变化 | 展示近12个月趋势 |
| M3.2 产品推荐 | 基于企业画像推荐产品 | 展示相关度>70%的产品 |
| M5.3 AI辅助审批 | AI建议审批结果 | 辅助人工决策 |
| M7.2 自定义报表 | 自定义维度的数据报表 | 支持导出Excel |
| M8.2 消息中心 | 统一的用户消息中心 | 消息分类、已读未读 |

#### WON'T (本期不包含)

| 功能 | 原因 |
|-----|------|
| 移动端APP | 优先Web端，APP后续规划 |
| 区块链存证 | 技术成熟度不足 |
| 实时支付对接 | 对接银行系统复杂 |
| 供应链金融 | 业务模式待验证 |

## 4. 功能详细需求

### 4.1 模块M1: 企业信用查询

```
需求编号: M1
需求名称: 企业信用查询与数据检索
优先级: MUST

功能描述:
- 支持按统一社会信用代码精确查询
- 支持按企业名称模糊搜索
- 支持多维度筛选(地区、行业、注册资本)

技术要求:
- 查询响应时间 < 2秒
- 数据源: 第三方工商API + 本地缓存
- 支持高并发: 1000 QPS

数据字段:
| 字段 | 类型 | 说明 |
|-----|------|------|
| company_name | string | 企业名称 |
| credit_code | string | 统一社会信用代码 |
| legal_person | string | 法定代表人 |
| registered_capital | decimal | 注册资本 |
| establishment_date | date | 成立日期 |
| business_status | string | 经营状态 |
| industry | string | 所属行业 |
| credit_score | int | 信用评分(0-100) |
| risk_level | string | 风险等级 |
```

### 4.2 模块M4: 贷款申请流程

```
需求编号: M4
需求名称: 贷款产品申请与材料提交
优先级: MUST

功能描述:
- 用户选择贷款产品并发起申请
- 填写企业信息、贷款金额、期限等
- 上传营业执照、财务报表等材料
- 实时保存申请进度

申请流程:
1. 选择产品 → 2. 填写信息 → 3. 上传材料 → 4. 提交申请 → 5. 初审

数据模型:
LoanApplication {
  application_id: string (UUID)
  user_id: string
  product_id: string
  company_name: string
  credit_code: string
  loan_amount: decimal
  loan_term: int (月)
  purpose: string
  status: enum [draft, submitted, reviewing, approved, rejected]
  created_at: datetime
  updated_at: datetime
}
```

### 4.3 模块M5: 自动化审批

```
需求编号: M5
需求名称: 贷款审批流程
优先级: MUST

审批流程:
[提交申请] → [自动预审] → [规则审核] → [人工复审] → [终审] → [结果通知]

规则引擎配置:
| 规则ID | 规则名称 | 条件 | 动作 |
|-------|---------|------|------|
| R001 | 企业状态检查 | business_status ≠ '存续' | 自动拒绝 |
| R002 | 信用分检查 | credit_score < 60 | 自动拒绝 |
| R003 | 额度限制 | loan_amount > 信用额度*1.5 | 标记待人工处理 |
| R004 | 行业限制 | industry in ['禁入行业'] | 自动拒绝 |
| R005 | 企业年龄 | establishment_date < 2年前 | 标记补充材料 |

审批状态流转:
draft → submitted → reviewing → [approved/rejected]
                      ↓
                needs_manual_review → manual_review → [approved/rejected]
```

### 4.4 模块M6: 风险评估

```
需求编号: M6
需求名称: 风险评估与评级系统
优先级: MUST

评分模型 (满分100分):

| 维度 | 权重 | 指标 |
|-----|------|------|
| 基础资质 | 30% | 成立年限、注册资本、经营状态 |
| 信用记录 | 35% | 信用评分、历史逾期、诉讼记录 |
| 财务状况 | 25% | 营收规模、资产负债、现金流 |
| 行业风险 | 10% | 行业周期、政策影响、竞争状况 |

风险等级定义:
| 等级 | 分数范围 | 说明 | 审批策略 |
|-----|---------|------|---------|
| AAA | 90-100 | 极低风险 | 自动通过 |
| AA | 80-89 | 低风险 | 自动通过 |
| A | 70-79 | 较低风险 | 自动通过 |
| BBB | 60-69 | 中等风险 | 人工复审 |
| BB | 50-59 | 较高风险 | 人工复审 |
| B | 40-49 | 高风险 | 人工终审 |
| C | 0-39 | 极高风险 | 自动拒绝 |

风控预警规则:
- 企业信用分连续下降>10分 → 预警
- 行业政策重大变化 → 预警
- 贷款申请数量异常增长 → 预警
```

### 4.5 模块M7: 数据可视化

```
需求编号: M7
需求名称: 数据分析与可视化
优先级: SHOULD

仪表盘指标:

1. 运营概览
   - 今日申请量
   - 今日审批通过率
   - 今日放款金额
   - 待审批数量

2. 趋势分析
   - 近7天/30天申请量趋势图
   - 审批通过率趋势图
   - 贷款金额分布图

3. 风控监控
   - 风险等级分布饼图
   - 高风险企业预警列表
   - 异常行为监控

4. 统计分析
   - 按地区分布
   - 按行业分布
   - 按贷款产品分布

技术要求:
- 图表库: ECharts
- 数据刷新间隔: 实时
- 支持数据导出(Excel/PDF)
```

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 |
|-----|------|
| 页面加载时间 | < 3秒 |
| API响应时间(P99) | < 500ms |
| 并发用户数 | ≥ 500 |
| 系统可用性 | ≥ 99.9% |

### 5.2 安全需求

- 用户密码加密存储 (BCrypt)
- 敏感数据脱敏展示
- 操作日志全记录
- API接口鉴权 (JWT)
- 防止SQL注入/XSS攻击

### 5.3 合规需求

- 《网络安全法》合规
- 《个人信息保护法》合规
- 《金融数据安全分级指南》
- 审计日志保留≥3年
